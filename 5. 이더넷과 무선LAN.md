# 5-1. 네트워크의 기본 구성
+ 네트워크의 기본 구성은 레이어2스위치로 하나의 네트워크를 구성해 그 사이에서 데이터를 전송한다.
+ 이러한 네트워크가 여러개가 있고, 서로 다른 네트워크로 데이터를 전송할때는 레이어3스위치나 라우터를 이용한다.
+ 같은 네트워크 내에서 전송하는 프로토콜로 자주 이용하는 것이 이더넷과 무선LAN이다.
+ 레이터2 스위치는 이더넷으로 전송하는 데이터에 전혀 변경을 가하지 않는다. 

# 5-2. MAC 주소
+ 이더넷은 이더넷 인터페이스 간에 데이터를 전송하므로 이더넷 인터페이스를 특정해야하는데, 이를 위해서 MAC주소가 필요하다.
+ MAC주소는 총 48비트이며 선두 24비트는 OUI로 이더넷 인터페이스 제조사 식별코드이며, 뒤 24비트는 시리얼 넘버로 구성된다.
+ MAC 주소는 이더넷 인터페이스에 미리 할당되어 있다. 변경불가.

# 5-3. 이더넷 헤더
+ 이더넷 헤더는 3가지 정보가 있다. 목적지MAC주소, 출발지MAC주소, 타입코드.
+ 이중 가장 중요한 것은 MAC주소이며, 타입코드는 IPv4를 나타내는 0x0800이 가장 많다.
+ 전송 대상이 되는 데이터는 64바이트에서 1500바이트 사이로 정해져 있다.
+ 데이터 크기의 최대값을 MTU(Maximum Transmission Unit)라고 부르고, MTU를 넘는 크기의 데이터는 복수로 분할해서 전송하게 된다.

# 5-4. 토폴로지
+ 네트워크에서 기기를 연결하는 형태를 나타내는 말.
+ 버스형, 스타형, 링형
+ 과거에는 동축케이블을 전송매체로하는 버스형 포톨로지가 많았지만, 현재는 레이어2스위치를 중심으로 하는 스타형이 대세이다.
+ 버스형 포톨로지는 하나의 전송매체를 복수의 기기가 공유하므로 어떤 순간에 데이터를 전송할 수 있는 것은 단 한 대뿐이다.
+ 그래서 어떻게 전송매체를 공유할지를 제어하는 것으로 CSMA/CD를 이용한다.
+ CSMA/CD는 우선 케이블이 현재 사용중인지를 감지한다. 비어있다면 바로 데이터를 전송하지 않고 랜덤시간동안 대기를 한 후 데이터를 전송한다.

# 5-5. 레이어2 스위치
+ 레이어2 스위치는 이더넷을 이용한 네트워크 하나를 구성하는 네트워크 기기이다.
+ 레이어2 스위치를 여러대 연결해도 하나의 네트워크이다.
+ 레이어2 스위치는 수신한 이더넷 프레임을 전혀 변경하지 않고 전송한다. 이더넷 헤더의 MAC주소를 확인할 뿐이다.

# 5-6, 레이어2 스위치 동작원리
+ 수신한 이더넷 프레임의 출발지 MAC주소를 MAC주소 테이블에 등록한다.
+ 목적지 MAC주소와 MAC주소 테이블에서 전송할 포트를 결정해 이더넷 프레임을 전송한다.
+ 만약 MAC주소 테이블에 존재하지 않는 목적지 MAC주소라면 수신 포트를 제외하고 모든 포트로 이더넷 프레임을 복사해 전송한다(플러딩)
+ 다른 포트로 플러딩 된다면 이더넷프레임은 파기되어 버리고, 그렇지 않고 다음 레이어2스위치를 만나면 다시 MAC테이블에 출발지MAC주소와 포트를 등록한다. 이를 반복
+ 통신은 양방향이므로 반대로 마찬가지로 MAC테이블에 등록과 동시에 목적지MAC주소와 같은 것이 등록되어 있으면 그 포트로 전송하면 된다.

# 5-7. 데이터 전송과 동시에 수신한다.
+ 레이어2 스위치를 기반으로 만든 이더넷 네트워크는 데이터 송수신을 동시에 할 수 있다.
+ 이를 전이중통신이라 부르며, 버스형 토폴로지같은 송신과 수신을 동시에 할 수 없는 것을 반이중통신이라고 한다.
+ 전이중통신을 실현하기 위한 방식은 데이터 수신용과 송신용으로 전송매체를 나누어 사용하는 것이다.
+ 레이어2 스위치와 PC의 이더넷 인터페이스는 UTP케이블로 연결되어 있는데 이 케이블의 속에는 4개의 줄이 들어있습니다. 이중 일부는 수신용, 일부는 송신용입니다.

# 5-8. 무선LAN
+ 무선LAN에서는 케이블 배선 없이 간편하게 네트워크를 만들 수 있다.
+ 무선LAN 액세스 포인트와 무선LAN 클라이언트로 무선LAN 네트워크를 구성한다. 
+ 무선LAN 액세스 포인트는 유선 이더넷에도 접속 가능하다.
+ 무선LAN은 주로 이용하는 전파의 주파수대와 데이터를 전파에 싣는 방식에 따라 여러가지 규격이 있다.
+ 무선LAN 통신을 위해서는 먼저 무선LAN 액세스포인트에 연결해서 무선LAN 링크를 확립해야 한다. 이를 어소시에이션이라고 부른다.
+ 어소시에이션은 SSID(Service Set Identifier)가 필요하다. 
+ 무선LAN 클라이언트는 무선LAN 액세스 포인트에서 내보내는 제어신호(비콘)에서 이용할 수 있는 주파수(채널)을 찾고 찾으면 SSID를 지정해서 어소시에이션 요청을 한다.

# 5-9. 전파는 돌려쓴다.
+ 무선LAN에서는 하나의 액세스 포인트에 접속하는 무선LAN클라이언트가 전파를 돌려쓴다. 따라서 규격의 속도만큼 나오지 않는데 실질적인 속도를 실효속도 및 스루풋이라고 부른다.
+ 무선 LAN에서 충돌이 발생하지 않게 하고, 전파를 돌려쓰도록 제어하고자 CSMA/CA를 이용한다.
+ 이유는 전파에 실어 송신할 수 있는 것은 하나의 무선LAN 클라이언트 뿐이기 때문이다.

# 5-10. CSMA/CA
+ 1. 전파가 이용인지 확인한다. 이용중이라면 대기를하고, 그렇지 않다면 랜덤시간 대기를 한다.
+ 2. 랜덤시간 대기를 하여도 전파가 미사용이라면 데이터를 전파에 실어 송신한다. 이에 대한 응답으로 ACK를 반환받는다.

# 5-11. 무선LAN의 보안
+ 무선LAN은 간편하고 편리한 만큼 악의적으로 이용될 수 있다.
+ 무선랜 보안의 핵심은 인증과 암호화이다.
+ 인증으로 무선LAN 액세스 포인트에 정식 사용자만 접속할 수 있다.
+ 암호화함으로써 무선LAN으로 송수신하는 데이터를 다른 사람이 엿보는 것을 방지한다.
+ 무선랜 보안규격으로는 WPA2(IEEE802.11i)가 있다.
