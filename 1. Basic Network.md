# 1-1. 네트워크는 무엇인가
+ 네트워크는 넓은 의미에서 물류, 교통, 인맥 등을 포함.
+ 여기서는 컴퓨터끼리 데이터를 주고받는 시스템인 컴퓨터 네트워크
+ 컴퓨터 네트워크는 pc나 스마트폰과 같은 정보 단말 기기를 서로 연결해서 만들어짐.
+ 컴퓨터 네트워크 덕분에 다른 사람과 데이터를 주고받을 수 있음.

# 1-2. 네트워크를 이용하는 목적
+ 정보를 수집
+ 문서 파일 등을 공유
+ 효율적으로 커뮤니케이션
+ 업무를 처리

# 1-3. 네트워크의 분류
+ 사설 네트워크 : 사내 네트워크나 가정 네트워크처럼 접속할 수 있는 사용자를 제한
+ 인터넷 : 접속할 사용자를 제한하지 않고, 누구나 이용할 수 있음.


# 1-4. LAN 과 WAN
+ LAN : 가정 내 네트워크 같은 각 거점마다의 구축된 네트워크를 말함
+ WAN : 그러한 LAN 들이 연결한 것이 WAN

# 1-5. 인터넷의 구성요소
+ 인터넷은 다양한 조직의 네트워크인 AS가 서로 연결되어 있음. AS의 예로는 구글이나 아마존.
+ AS는 인터넷 접속 서비스를 제공하는 ISP이다.
+ ISP의 상위 그룹은 Tier1 이다. Tier1 이외의 ISP는 최종적으로 Tier1에 연결되어 자신이 관리하지 않는 네트워크의 정보도 입수. 따라서 인터넷 상의 모든 ISP는 Tier1을 경유해서 연결. 이말을 즉 자신이 계약한 ISP의 사용자뿐만 아니라 다른 ISP의 사용자와도 통신할 수 있음.

# 1-6. 데이터를 주고받는 주체
+ 데이터는 서버와 클라이언트에서 주고받는다. 이는 클라이언트 서버 애플리케이션이라고 부른다.
+ 서버를 거치지 않고 클라이언트끼지 직접 데이터를 주고받는 애플리케이션을 피어투피어 애플리케이션이라고 부른다.

# 1-7. 통신을 위한 규칙
+ 한국어나 영어처럼 언어를 컴퓨터에서는 네트워크 아키텍처라고 한다. 언어에서 문법과 발음같은 규칙들이 있듯이 네트워크 아키텍처에서도 데이터 형식이나 통신절차 같은 규칙들을 프로토콜이라고 한다. 이러한 프로코톨의 집합이 바로 네트워크 아키텍처이다.
