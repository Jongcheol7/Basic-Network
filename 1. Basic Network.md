# 1-1. 네트워크는 무엇인가
+ 네트워크는 넓은 의미에서 물류, 교통, 인맥 등을 포함.
+ 여기서는 컴퓨터끼리 데이터를 주고받는 시스템인 컴퓨터 네트워크
+ 컴퓨터 네트워크는 pc나 스마트폰과 같은 정보 단말 기기를 서로 연결해서 만들어짐.
+ 컴퓨터 네트워크 덕분에 다른 사람과 데이터를 주고받을 수 있음.

# 1-2. 네트워크를 이용하는 목적
+ 정보를 수집
+ 문서 파일 등을 공유
+ 효율적으로 커뮤니케이션
+ 업무를 처리

# 1-3. 네트워크의 분류
+ 사설 네트워크 : 사내 네트워크나 가정 네트워크처럼 접속할 수 있는 사용자를 제한
+ 인터넷 : 접속할 사용자를 제한하지 않고, 누구나 이용할 수 있음.


# 1-4. LAN 과 WAN
+ LAN : 가정 내 네트워크 같은 각 거점마다의 구축된 네트워크를 말함
+ WAN : 그러한 LAN 들이 연결한 것이 WAN

# 1-5. 인터넷의 구성요소
+ 인터넷은 다양한 조직의 네트워크인 AS가 서로 연결되어 있음. AS의 예로는 구글이나 아마존.
+ AS는 인터넷 접속 서비스를 제공하는 ISP이다.
+ ISP의 상위 그룹은 Tier1 이다. Tier1 이외의 ISP는 최종적으로 Tier1에 연결되어 자신이 관리하지 않는 네트워크의 정보도 입수. 따라서 인터넷 상의 모든 ISP는 Tier1을 경유해서 연결. 이말을 즉 자신이 계약한 ISP의 사용자뿐만 아니라 다른 ISP의 사용자와도 통신할 수 있음.

# 1-6. 데이터를 주고받는 주체
+ 데이터는 서버와 클라이언트에서 주고받는다. 이는 클라이언트 서버 애플리케이션이라고 부른다.
+ 서버를 거치지 않고 클라이언트끼지 직접 데이터를 주고받는 애플리케이션을 피어투피어 애플리케이션이라고 부른다.

# 1-7. 통신을 위한 규칙
+ 한국어나 영어처럼 언어를 컴퓨터에서는 네트워크 아키텍처라고 한다. 언어에서 문법과 발음같은 규칙들이 있듯이 네트워크 아키텍처에서도 데이터 형식이나 통신절차 같은 규칙들을 프로토콜이라고 한다. 이러한 프로코톨의 집합이 바로 네트워크 아키텍처이다.

# 1-8. 네트워크의 공통 언어 TCP/IP
+ 네트워크 아키텍처에는 여러 종류가 있지만 현재는 TCP/IP를 이용한다. TCP/IP는 네트워크를 통해 애플리케이션의 데이터를 주고받기 위해, 역할별로 4가지로 계층화된 복수 프로토콜을 조합한다. 애플리케이션층(HTTP,SMTP,POP3,IMAP4,DHCP,DNS), 트랜스포트층(TCP/UDP), 인터넷층(IP,ICMP,ARP), 네트워크 인터페이스층(이더넷, 무선LAN)

# 1-9. 서버 운용.관리의 어려운점
+ 애플리케이션을 제대로 동작하려면 서버가 항상 가동되어 있어야 한다. 그리고 적절한 하드웨어를 선정해야 하며, 서버의 상태를 항상 감시하고, 문제가 생기면 바로 대처해야한다. 백업도 해두어야하고, 보안대책도 중요한 만큼 시간과 비용이 발생.
+ 위처럼 직접 서버를 관리하는 방법은 온프레미스라고 한다.

# 1-10. 클라우드 서비스
+ 서버를 직접 관리 운용하지 않고, 인터넷을 통해 서버의 기능만 이용하는 것을 클라우드 서비스라고 한다. 
+ 클라우드 서비스는 매우 편리하지만 보안이나 가용성에 주의를 해야한다. 보안은 자신이 관리가 미치지 않은 범위에서 데이터가 유지되기 때문이고, 서비스를 이용할 수 없게 될 가능성에 있다는 점에서 가용성에 주의해야 한다.

# 1-11. 클라우드 서비스의 분류
+ 클라우드 서비스는 네트워크를 경유해 서버의 어느 부분을 사용자가 이용할 수 있게 하느냐에 따라 분류할 수 있다.
+ IaaS : 네트워크를 경유해 서버의 CPU와 메모리, 저장장치와 같은 하드웨어 부분을 이용. 사용자는 IaaS의 서버상에서 OS와 미들웨어, 애플리케이션을 추가한다. 
+ PaaS : 하드웨어에 더해 네트워크를 경유해 서버의 플랫폼을 이용할 수 있다. 플랫폼은 OS와 OS상에서 동작하는 데이터베이스 등을 가리킨다.
+ SaaS : 하드웨어 부분부터 애플리케이션 부분까지. 네트워크를 경유해 서버의 특정 소프트웨어 기능을 이용할 수 있게 한다. 일반 개인 사용자가 이용하는 클라우드 서비스는 대부분 SaaS이다. 예를들어 온라인 스토리지 서비스로 사용자는 자유롭게 파일을 저장하고 공유한다.
